Design Process

From Concept to Reality

1. Research & Discovery

Analyzed my own pain points with existing budget apps. Identified core use cases: quick expense logging, income tracking, savings goals, and spending insights. Mapped out the daily workflow I wanted.

2. Information Architecture

Designed a clear navigation structure: Dashboard for overview, separate sections for Expenses/Income/Savings, Budgets for planning, Reports for analysis, and Settings for customization.

3. Visual Design

Created a dark theme UI with vibrant accent colors for data visualization. Focused on readability, hierarchy, and reducing visual clutter. Used color strategically: green for income, red for expenses, purple for savings.

4. Development & Iteration

Built with Next.js and Supabase for a modern, performant stack. Continuously refined the UX based on actual daily usage ‚Äî adding quick-add features, improving date filtering, simplifying forms.

Challenges & Solutions

Problems I Solved

Challenge

Date Filtering Accuracy

Transactions weren't appearing in "This Month" filter due to timezone issues when parsing ISO date strings.

Solution

Implemented local timezone-aware date parsing by splitting YYYY-MM-DD strings and constructing Date objects with explicit year, month, day values.

Challenge

Quick Category Creation

Users (me) often needed to create new categories while adding an expense, but had to navigate away to settings first.

Solution

Added "Add New" buttons inline with each dropdown in the expense form, opening a quick modal to create new items without leaving the context.

Challenge

Database Schema Mismatch

Supabase uses snake_case columns while the frontend code uses camelCase, causing data mapping issues.

Solution

Created bidirectional utility functions to automatically convert between snake_case and camelCase when reading from and writing to the database.

Challenge

Real-time Auth State

Managing authentication state consistently across server components, client components, and API routes in Next.js 15.

Solution

Implemented separate Supabase client factories for browser, server, and middleware contexts using @supabase/ssr package with proper cookie handling.
